\doxysubsection{Detailed Description}
The abstract base class for all objects that form the layout system. 

This is an abstract base class. As such, it can\textquotesingle{}t be instantiated directly, rather use one of its subclasses.

A Layout element is a rectangular object which can be placed in layouts. It has an outer rect (\doxylink{class_q_c_p_layout_element_a2a32a12a6161c9dffbadeb9cc585510c_a2a32a12a6161c9dffbadeb9cc585510c}{QCPLayout\+Element\+::outer\+Rect}) and an inner rect (\doxylink{class_q_c_p_layout_element_a208effccfe2cca4a0eaf9393e60f2dd4_a208effccfe2cca4a0eaf9393e60f2dd4}{QCPLayout\+Element\+::rect}). The difference between outer and inner rect is called its margin. The margin can either be set to automatic or manual (\doxylink{class_q_c_p_layout_element_accfda49994e3e6d51ed14504abf9d27d_accfda49994e3e6d51ed14504abf9d27d}{set\+Auto\+Margins}) on a per-\/side basis. If a side is set to manual, that margin can be set explicitly with \doxylink{class_q_c_p_layout_element_a8f450b1f3f992ad576fce2c63d8b79cf_a8f450b1f3f992ad576fce2c63d8b79cf}{set\+Margins} and will stay fixed at that value. If it\textquotesingle{}s set to automatic, the layout element subclass will control the value itself (via \doxylink{class_q_c_p_layout_element_a005c9f0fe84bc1591a2cf2c46fd477b4_a005c9f0fe84bc1591a2cf2c46fd477b4}{calculate\+Auto\+Margin}).

Layout elements can be placed in layouts (base class \doxylink{class_q_c_p_layout}{QCPLayout}) like \doxylink{class_q_c_p_layout_grid}{QCPLayout\+Grid}. The top level layout is reachable via \doxylink{class_q_custom_plot_af1a1f1f571237deb7c2bd34a5e9f018f_af1a1f1f571237deb7c2bd34a5e9f018f}{QCustom\+Plot\+::plot\+Layout}, and is a \doxylink{class_q_c_p_layout_grid}{QCPLayout\+Grid}. Since \doxylink{class_q_c_p_layout}{QCPLayout} itself derives from \doxylink{class_q_c_p_layout_element}{QCPLayout\+Element}, layouts can be nested.

Thus in \doxylink{class_q_custom_plot}{QCustom\+Plot} one can divide layout elements into two categories\+: The ones that are invisible by themselves, because they don\textquotesingle{}t draw anything. Their only purpose is to manage the position and size of other layout elements. This category of layout elements usually use \doxylink{class_q_c_p_layout}{QCPLayout} as base class. Then there is the category of layout elements which actually draw something. For example, \doxylink{class_q_c_p_axis_rect}{QCPAxis\+Rect}, \doxylink{class_q_c_p_legend}{QCPLegend} and \doxylink{class_q_c_p_text_element}{QCPText\+Element} are of this category. This does not necessarily mean that the latter category can\textquotesingle{}t have child layout elements. \doxylink{class_q_c_p_legend}{QCPLegend} for instance, actually derives from \doxylink{class_q_c_p_layout_grid}{QCPLayout\+Grid} and the individual legend items are child layout elements in the grid layout.

This event gets called when the user turns the mouse scroll wheel while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{class_q_c_p_layout_element_a62d60f1b0ee9bb23e38811d71586d79c_a62d60f1b0ee9bb23e38811d71586d79c}{select\+Test}.

The {\ttfamily event-\/\texorpdfstring{$>$}{>}angle\+Delta()} indicates how far the mouse wheel was turned, which is usually +/-\/ 120 for single rotation steps. However, if the mouse wheel is turned rapidly, multiple steps may accumulate to one event, making the delta larger. On the other hand, if the wheel has very smooth steps or none at all, the delta may be smaller.

The default implementation does nothing.
