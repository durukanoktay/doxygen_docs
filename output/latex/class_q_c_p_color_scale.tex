\doxysubsection{Detailed Description}
A color scale for use with color coding data such as \doxylink{class_q_c_p_color_map}{QCPColor\+Map}. 

This layout element can be placed on the plot to correlate a color gradient with data values. It is usually used in combination with one or multiple \doxylink{class_q_c_p_color_map}{QCPColor\+Maps}.



The color scale can be either horizontal or vertical, as shown in the image above. The orientation and the side where the numbers appear is controlled with \doxylink{class_q_c_p_color_scale_a1bf9bdb291927c422dd66b404b206f1f_a1bf9bdb291927c422dd66b404b206f1f}{set\+Type}.

Use \doxylink{class_q_c_p_color_map_aa828921db364fe3c6af4619580ab85fd_aa828921db364fe3c6af4619580ab85fd}{QCPColor\+Map\+::set\+Color\+Scale} to connect a color map with a color scale. Once they are connected, they share their gradient, data range and data scale type (\doxylink{class_q_c_p_color_scale_a1f29583bb6f1e7f473b62fb712be3940_a1f29583bb6f1e7f473b62fb712be3940}{set\+Gradient}, \doxylink{class_q_c_p_color_scale_abe88633003a26d1e756aa74984587fef_abe88633003a26d1e756aa74984587fef}{set\+Data\+Range}, \doxylink{class_q_c_p_color_scale_aeb6107d67dd7325145b2498abae67fc3_aeb6107d67dd7325145b2498abae67fc3}{set\+Data\+Scale\+Type}). Multiple color maps may be associated with a single color scale, to make them all synchronize these properties.

To have finer control over the number display and axis behaviour, you can directly access the \doxylink{class_q_c_p_color_scale_a39bdbdb3b212602a5a57f9f3ea444190_a39bdbdb3b212602a5a57f9f3ea444190}{axis}. Returns the inner rect of this layout element. The inner rect is the outer rect (\doxylink{class_q_c_p_layout_element_a2a32a12a6161c9dffbadeb9cc585510c_a2a32a12a6161c9dffbadeb9cc585510c}{outer\+Rect}, \doxylink{class_q_c_p_layout_element_a38975ea13e36de8e53391ce41d94bc0f_a38975ea13e36de8e53391ce41d94bc0f}{set\+Outer\+Rect}) shrinked by the margins (\doxylink{class_q_c_p_layout_element_a8f450b1f3f992ad576fce2c63d8b79cf_a8f450b1f3f992ad576fce2c63d8b79cf}{set\+Margins}, \doxylink{class_q_c_p_layout_element_accfda49994e3e6d51ed14504abf9d27d_accfda49994e3e6d51ed14504abf9d27d}{set\+Auto\+Margins}).

In some cases, the area between outer and inner rect is left blank. In other cases the margin area is used to display peripheral graphics while the main content is in the inner rect. void QCPColor\+Scale\+::wheel\+Event (\begin{DoxyParamCaption}\item[{QWheel\+Event \texorpdfstring{$\ast$}{*}}]{event}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}}

This event gets called when the user turns the mouse scroll wheel while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{class_q_c_p_layout_element_a62d60f1b0ee9bb23e38811d71586d79c_a62d60f1b0ee9bb23e38811d71586d79c}{select\+Test}.

The current pixel position of the cursor on the \doxylink{class_q_custom_plot}{QCustom\+Plot} widget is accessible via {\ttfamily event-\/\texorpdfstring{$>$}{>}pos()}.

The {\ttfamily event-\/\texorpdfstring{$>$}{>}angle\+Delta()} indicates how far the mouse wheel was turned, which is usually +/-\/ 120 for single rotation steps. However, if the mouse wheel is turned rapidly, multiple steps may accumulate to one event, making the delta larger. On the other hand, if the wheel has very smooth steps or none at all, the delta may be smaller.

The default implementation does nothing.
