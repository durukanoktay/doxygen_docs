\doxysubsection{Detailed Description}
The abstract base class for all data representing objects in a plot. 

It defines a very basic interface like name, pen, brush, visibility etc. Since this class is abstract, it can\textquotesingle{}t be instantiated. Use one of the subclasses or create a subclass yourself to create new ways of displaying data (see "{}\+Creating own plottables"{} below). Plottables that display one-\/dimensional data (i.\+e. data points have a single key dimension and one or multiple values at each key) are based off of the template subclass \doxylink{class_q_c_p_abstract_plottable1_d}{QCPAbstract\+Plottable1D}, see details there.

All further specifics are in the subclasses, for example\+: \begin{DoxyItemize}
\item A normal graph with possibly a line and/or scatter points \doxylink{class_q_c_p_graph}{QCPGraph} (typically created with \doxylink{class_q_custom_plot_a2836a46e31d5aee174ae054c0aa6580b_a2836a46e31d5aee174ae054c0aa6580b}{QCustom\+Plot\+::add\+Graph}) \item A parametric curve\+: \doxylink{class_q_c_p_curve}{QCPCurve} \item A bar chart\+: \doxylink{class_q_c_p_bars}{QCPBars} \item A statistical box plot\+: \doxylink{class_q_c_p_statistical_box}{QCPStatistical\+Box} \item A color encoded two-\/dimensional map\+: \doxylink{class_q_c_p_color_map}{QCPColor\+Map} \item An OHLC/\+Candlestick chart\+: \doxylink{class_q_c_p_financial}{QCPFinancial}\end{DoxyItemize}
\hypertarget{class_q_c_p_abstract_plottable_plottables-subclassing}{}\doxysubsection{\texorpdfstring{Creating own plottables}{Creating own plottables}}\label{class_q_c_p_abstract_plottable_plottables-subclassing}
Subclassing directly from \doxylink{class_q_c_p_abstract_plottable}{QCPAbstract\+Plottable} is only recommended if you wish to display two-\/dimensional data like \doxylink{class_q_c_p_color_map}{QCPColor\+Map}, i.\+e. two logical key dimensions and one (or more) data dimensions. If you want to display data with only one logical key dimension, you should rather derive from \doxylink{class_q_c_p_abstract_plottable1_d}{QCPAbstract\+Plottable1D}.

If subclassing \doxylink{class_q_c_p_abstract_plottable}{QCPAbstract\+Plottable} directly, these are the pure virtual functions you must implement\+: \begin{DoxyItemize}
\item \doxylink{class_q_c_p_abstract_plottable_a8e4964b629802ce50e1427c7fb21d35b_a8e4964b629802ce50e1427c7fb21d35b}{select\+Test} \item \doxylink{class_q_c_p_abstract_plottable_a453f676a5cee7bf846c5f0fa05ea84b3_a453f676a5cee7bf846c5f0fa05ea84b3}{draw} \item \doxylink{class_q_c_p_abstract_plottable_a71c259f4f96533d12df152c446b17425_a71c259f4f96533d12df152c446b17425}{draw\+Legend\+Icon} \item \doxylink{class_q_c_p_abstract_plottable_a56b4d9000f4de0de86c566c84eb05a21_a56b4d9000f4de0de86c566c84eb05a21}{get\+Key\+Range} \item \doxylink{class_q_c_p_abstract_plottable_aa81c95aec08d86cdd5407e48b83ab186_aa81c95aec08d86cdd5407e48b83ab186}{get\+Value\+Range}\end{DoxyItemize}
See the documentation of those functions for what they need to do.

For drawing your plot, you can use the \doxylink{class_q_c_p_abstract_plottable_a7ad84a36472441cf1f555c5683d0da93_a7ad84a36472441cf1f555c5683d0da93}{coords\+To\+Pixels} functions to translate a point in plot coordinates to pixel coordinates. This function is quite convenient, because it takes the orientation of the key and value axes into account for you (x and y are swapped when the key axis is vertical and the value axis horizontal). If you are worried about performance (i.\+e. you need to translate many points in a loop like \doxylink{class_q_c_p_graph}{QCPGraph}), you can directly use \doxylink{class_q_c_p_axis_af15d1b3a7f7e9b53d759d3ccff1fe4b4_af15d1b3a7f7e9b53d759d3ccff1fe4b4}{QCPAxis\+::coord\+To\+Pixel}. However, you must then take care about the orientation of the axis yourself.

This event gets called when the user presses the mouse button a second time in a double-\/click, while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{class_q_c_p_abstract_plottable_a8e4964b629802ce50e1427c7fb21d35b_a8e4964b629802ce50e1427c7fb21d35b}{select\+Test}.

So in the case of a double-\/click, the event succession is {\itshape press\+Event -- release\+Event -- double\+Click\+Event -- release\+Event}.

The current pixel position of the cursor on the \doxylink{class_q_custom_plot}{QCustom\+Plot} widget is accessible via {\ttfamily event-\/\texorpdfstring{$>$}{>}pos()}. The parameter {\itshape details} contains layerable-\/specific details about the hit, which were generated in the previous call to \doxylink{class_q_c_p_abstract_plottable_a8e4964b629802ce50e1427c7fb21d35b_a8e4964b629802ce50e1427c7fb21d35b}{select\+Test}. Multi-\/part objects convey the specific {\ttfamily Selectable\+Part} that was hit (e.\+g. \doxylink{class_q_c_p_axis_abee4c7a54c468b1385dfce2c898b115f_abee4c7a54c468b1385dfce2c898b115f}{QCPAxis\+::\+Selectable\+Part} in the case of axes).

Similarly to \doxylink{class_q_c_p_layerable_af6567604818db90f4fd52822f8bc8376_af6567604818db90f4fd52822f8bc8376}{mouse\+Press\+Event}, once a layerable has accepted the \doxylink{class_q_c_p_layerable_a4171e2e823aca242dd0279f00ed2de81_a4171e2e823aca242dd0279f00ed2de81}{mouse\+Double\+Click\+Event}, it is considered the mouse grabber and will receive all following calls to \doxylink{class_q_c_p_layerable_a9eee1ba47fd69be111059ca3881933e4_a9eee1ba47fd69be111059ca3881933e4}{mouse\+Move\+Event} and \doxylink{class_q_c_p_layerable_aa0d79b005686f668622bbe66ac03ba2c_aa0d79b005686f668622bbe66ac03ba2c}{mouse\+Release\+Event} for this mouse interaction (a "{}mouse interaction"{} in this context ends with the release).

void QCPAbstract\+Plottable\+::rescale\+Axes (\begin{DoxyParamCaption}\item[{bool}]{only\+Enlarge}{ = {\ttfamily false}}\end{DoxyParamCaption}) const}

Rescales the key and value axes associated with this plottable to contain all displayed data, so the whole plottable is visible. If the scaling of an axis is logarithmic, rescale\+Axes will make sure not to rescale to an illegal range i.\+e. a range containing different signs and/or zero. Instead it will stay in the current sign domain and ignore all parts of the plottable that lie outside of that domain.

{\itshape only\+Enlarge} makes sure the ranges are only expanded, never reduced. So it\textquotesingle{}s possible to show multiple plottables in their entirety by multiple calls to rescale\+Axes where the first call has {\itshape only\+Enlarge} set to false (the default), and all subsequent set to true.

{\footnotesize\ttfamily \label{class_q_c_p_abstract_plottable_a8e4964b629802ce50e1427c7fb21d35b_a8e4964b629802ce50e1427c7fb21d35b} 
virtual double QCPAbstract\+Plottable\+::select\+Test (\begin{DoxyParamCaption}\item[{const QPointF \&}]{pos}{, }\item[{bool}]{only\+Selectable}{, }\item[{QVariant \texorpdfstring{$\ast$}{*}}]{details}{ = {\ttfamily nullptr}}\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [pure virtual]}}

This function is used to decide whether a click hits a layerable object or not.

{\itshape pos} is a point in pixel coordinates on the \doxylink{class_q_custom_plot}{QCustom\+Plot} surface. This function returns the shortest pixel distance of this point to the object. If the object is either invisible or the distance couldn\textquotesingle{}t be determined, -\/1.\+0 is returned. Further, if {\itshape only\+Selectable} is true and the object is not selectable, -\/1.\+0 is returned, too.

If the object is represented not by single lines but by an area like a \doxylink{class_q_c_p_item_text}{QCPItem\+Text} or the bars of a \doxylink{class_q_c_p_bars}{QCPBars} plottable, a click inside the area should also be considered a hit. In these cases this function thus returns a constant value greater zero but still below the parent plot\textquotesingle{}s selection tolerance. (typically the selection\+Tolerance multiplied by 0.\+99).

Providing a constant value for area objects allows selecting line objects even when they are obscured by such area objects, by clicking close to the lines (i.\+e. closer than 0.\+99\texorpdfstring{$\ast$}{*}selection\+Tolerance).

The actual setting of the selection state is not done by this function. This is handled by the parent \doxylink{class_q_custom_plot}{QCustom\+Plot} when the mouse\+Release\+Event occurs, and the finally selected object is notified via the select\+Event/ deselect\+Event methods.

{\itshape details} is an optional output parameter. Every layerable subclass may place any information in {\itshape details}. This information will be passed to \doxylink{class_q_c_p_abstract_plottable_a2d488568cf16600dd81fa23d7d439829_a2d488568cf16600dd81fa23d7d439829}{select\+Event} when the parent \doxylink{class_q_custom_plot}{QCustom\+Plot} decides on the basis of this select\+Test call, that the object was successfully selected. The subsequent call to \doxylink{class_q_c_p_abstract_plottable_a2d488568cf16600dd81fa23d7d439829_a2d488568cf16600dd81fa23d7d439829}{select\+Event} will carry the {\itshape details}. This is useful for multi-\/part objects (like \doxylink{class_q_c_p_axis}{QCPAxis}). This way, a possibly complex calculation to decide which part was clicked is only done once in \doxylink{class_q_c_p_abstract_plottable_a8e4964b629802ce50e1427c7fb21d35b_a8e4964b629802ce50e1427c7fb21d35b}{select\+Test}. The result (i.\+e. the actually clicked part) can then be placed in {\itshape details}. So in the subsequent \doxylink{class_q_c_p_abstract_plottable_a2d488568cf16600dd81fa23d7d439829_a2d488568cf16600dd81fa23d7d439829}{select\+Event}, the decision which part was selected doesn\textquotesingle{}t have to be done a second time for a single selection operation.

In the case of 1D Plottables (\doxylink{class_q_c_p_abstract_plottable1_d}{QCPAbstract\+Plottable1D}, like \doxylink{class_q_c_p_graph}{QCPGraph} or \doxylink{class_q_c_p_bars}{QCPBars}) {\itshape details} will be set to a \doxylink{class_q_c_p_data_selection}{QCPData\+Selection}, describing the closest data point to {\itshape pos}.

You may pass {\ttfamily nullptr} as {\itshape details} to indicate that you are not interested in those selection details.

void QCPLayerable\+::wheel\+Event (\begin{DoxyParamCaption}\item[{QWheel\+Event \texorpdfstring{$\ast$}{*}}]{event}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}, {\ttfamily [inherited]}}

This event gets called when the user turns the mouse scroll wheel while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{class_q_c_p_abstract_plottable_a8e4964b629802ce50e1427c7fb21d35b_a8e4964b629802ce50e1427c7fb21d35b}{select\+Test}.

The current pixel position of the cursor on the \doxylink{class_q_custom_plot}{QCustom\+Plot} widget is accessible via {\ttfamily event-\/\texorpdfstring{$>$}{>}pos()}.

The {\ttfamily event-\/\texorpdfstring{$>$}{>}angle\+Delta()} indicates how far the mouse wheel was turned, which is usually +/-\/ 120 for single rotation steps. However, if the mouse wheel is turned rapidly, multiple steps may accumulate to one event, making the delta larger. On the other hand, if the wheel has very smooth steps or none at all, the delta may be smaller.
