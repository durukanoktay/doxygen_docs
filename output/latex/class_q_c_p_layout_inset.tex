\doxysubsection{Detailed Description}
A layout that places child elements aligned to the border or arbitrarily positioned. 

Elements are placed either aligned to the border or at arbitrary position in the area of the layout. Which placement applies is controlled with the \doxylink{class_q_c_p_layout_inset_a8b9e17d9a2768293d2a7d72f5e298192_a8b9e17d9a2768293d2a7d72f5e298192}{Inset\+Placement} (\doxylink{class_q_c_p_layout_inset_a63298830744d5d8c5345511c00fd2144_a63298830744d5d8c5345511c00fd2144}{set\+Inset\+Placement}).

Elements are added via \doxylink{class_q_c_p_layout_inset_ad61529eb576af7f04dff94abb10c745a_ad61529eb576af7f04dff94abb10c745a}{add\+Element(\+QCPLayout\+Element \texorpdfstring{$\ast$}{*}element, Qt\+::\+Alignment alignment)} or \doxylink{class_q_c_p_layout_inset_a8ff61fbee4a1f0ff45c398009d9f1e56_a8ff61fbee4a1f0ff45c398009d9f1e56}{add\+Element(\+QCPLayout\+Element \texorpdfstring{$\ast$}{*}element, const QRect\+F \&rect)}. If the first method is used, the inset placement will default to \doxylink{class_q_c_p_layout_inset_a8b9e17d9a2768293d2a7d72f5e298192_a8b9e17d9a2768293d2a7d72f5e298192aa81e7df4a785ddee2229a8f47c46e817}{ip\+Border\+Aligned} and the element will be aligned according to the {\itshape alignment} parameter. The second method defaults to \doxylink{class_q_c_p_layout_inset_a8b9e17d9a2768293d2a7d72f5e298192_a8b9e17d9a2768293d2a7d72f5e298192aa4802986ea2cea457f932b115acba59e}{ip\+Free} and allows placing elements at arbitrary position and size, defined by {\itshape rect}.

The alignment or rect can be set via \doxylink{class_q_c_p_layout_inset_a62882a4f9ad58bb0f53da12fde022abe_a62882a4f9ad58bb0f53da12fde022abe}{set\+Inset\+Alignment} or \doxylink{class_q_c_p_layout_inset_aa487c8378a6f9533567a2e6430099dc3_aa487c8378a6f9533567a2e6430099dc3}{set\+Inset\+Rect}, respectively.

This is the layout that every \doxylink{class_q_c_p_axis_rect}{QCPAxis\+Rect} has as \doxylink{class_q_c_p_axis_rect_a949f803466619924c7018df4b511ae10_a949f803466619924c7018df4b511ae10}{QCPAxis\+Rect\+::inset\+Layout}.

This event gets called when the user presses the mouse button a second time in a double-\/click, while the cursor is over the layerable. Whether a cursor is over the layerable is decided by a preceding call to \doxylink{class_q_c_p_layout_inset_a1569969c3e7e92f64ea13d46616e5a8a_a1569969c3e7e92f64ea13d46616e5a8a}{select\+Test}.

The \doxylink{class_q_c_p_layerable_a4171e2e823aca242dd0279f00ed2de81_a4171e2e823aca242dd0279f00ed2de81}{mouse\+Double\+Click\+Event} is called instead of the second \doxylink{class_q_c_p_layerable_af6567604818db90f4fd52822f8bc8376_af6567604818db90f4fd52822f8bc8376}{mouse\+Press\+Event}. So in the case of a double-\/click, the event succession is {\itshape press\+Event -- release\+Event -- double\+Click\+Event -- release\+Event}.

The current pixel position of the cursor on the \doxylink{class_q_custom_plot}{QCustom\+Plot} widget is accessible via {\ttfamily event-\/\texorpdfstring{$>$}{>}pos()}. The parameter {\itshape details} contains layerable-\/specific details about the hit, which were generated in the previous call to \doxylink{class_q_c_p_layout_inset_a1569969c3e7e92f64ea13d46616e5a8a_a1569969c3e7e92f64ea13d46616e5a8a}{select\+Test}. For example, One-\/dimensional plottables like \doxylink{class_q_c_p_graph}{QCPGraph} or \doxylink{class_q_c_p_bars}{QCPBars} convey the clicked data point in the {\itshape details} parameter, as \doxylink{class_q_c_p_data_selection}{QCPData\+Selection} packed as QVariant. Multi-\/part objects convey the specific {\ttfamily Selectable\+Part} that was hit (e.\+g. \doxylink{class_q_c_p_axis_abee4c7a54c468b1385dfce2c898b115f_abee4c7a54c468b1385dfce2c898b115f}{QCPAxis\+::\+Selectable\+Part} in the case of axes).

Similarly to \doxylink{class_q_c_p_layerable_af6567604818db90f4fd52822f8bc8376_af6567604818db90f4fd52822f8bc8376}{mouse\+Press\+Event}, once a layerable has accepted the \doxylink{class_q_c_p_layerable_a4171e2e823aca242dd0279f00ed2de81_a4171e2e823aca242dd0279f00ed2de81}{mouse\+Double\+Click\+Event}, it is considered the mouse grabber and will receive all following calls to \doxylink{class_q_c_p_layerable_a9eee1ba47fd69be111059ca3881933e4_a9eee1ba47fd69be111059ca3881933e4}{mouse\+Move\+Event} and \doxylink{class_q_c_p_layerable_aa0d79b005686f668622bbe66ac03ba2c_aa0d79b005686f668622bbe66ac03ba2c}{mouse\+Release\+Event} for this mouse interaction (a "{}mouse interaction"{} in this context ends with the release).

The default implementation does nothing except explicitly ignoring the event with {\ttfamily event-\/\texorpdfstring{$>$}{>}ignore()}.
