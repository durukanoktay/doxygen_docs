\doxysubsection{Detailed Description}
A plottable representing a bar chart in a plot. 



To plot data, assign it with the \doxylink{class_q_c_p_bars_a6dc562ec7120a8521e1061f2134367e4_a6dc562ec7120a8521e1061f2134367e4}{set\+Data} or \doxylink{class_q_c_p_bars_a323d6970d6d6e3166d89916a7f60f733_a323d6970d6d6e3166d89916a7f60f733}{add\+Data} functions.\hypertarget{class_q_c_p_bars_qcpbars-appearance}{}\doxysubsection{\texorpdfstring{Changing the appearance}{Changing the appearance}}\label{class_q_c_p_bars_qcpbars-appearance}
The appearance of the bars is determined by the pen and the brush (\doxylink{class_q_c_p_abstract_plottable_ab74b09ae4c0e7e13142fe4b5bf46cac7_ab74b09ae4c0e7e13142fe4b5bf46cac7}{set\+Pen}, \doxylink{class_q_c_p_abstract_plottable_a7a4b92144dca6453a1f0f210e27edc74_a7a4b92144dca6453a1f0f210e27edc74}{set\+Brush}). The width of the individual bars can be controlled with \doxylink{class_q_c_p_bars_adcaa3b41281bb2c0f7949b341592fcc0_adcaa3b41281bb2c0f7949b341592fcc0}{set\+Width\+Type} and \doxylink{class_q_c_p_bars_afec6116579d44d5b706e0fa5e5332507_afec6116579d44d5b706e0fa5e5332507}{set\+Width}.

Bar charts are stackable. This means, two \doxylink{class_q_c_p_bars}{QCPBars} plottables can be placed on top of each other (see \doxylink{class_q_c_p_bars_ac22e00a6a41509538c21b04f0a57318c_ac22e00a6a41509538c21b04f0a57318c}{QCPBars\+::move\+Above}). So when two bars are at the same key position, they will appear stacked.

If you would like to group multiple \doxylink{class_q_c_p_bars}{QCPBars} plottables together so they appear side by side as shown below, use \doxylink{class_q_c_p_bars_group}{QCPBars\+Group}.

Rescales the key and value axes associated with this plottable to contain all displayed data, so the whole plottable is visible. If the scaling of an axis is logarithmic, rescale\+Axes will make sure not to rescale to an illegal range i.\+e. a range containing different signs and/or zero. So it\textquotesingle{}s possible to show multiple plottables in their entirety by multiple calls to rescale\+Axes where the first call has {\itshape only\+Enlarge} set to false (the default), and all subsequent set to true.

However, if the mouse wheel is turned rapidly, multiple steps may accumulate to one event, making the delta larger. On the other hand, if the wheel has very smooth steps or none at all, the delta may be smaller.

The default implementation does nothing.
